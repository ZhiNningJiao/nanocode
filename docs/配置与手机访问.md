# Nanocode 配置说明与手机访问

## 本机访问地址（当前环境）

nanocode 和代码都在 **Linux 服务器**上；你用 **Mac 上的 Cursor** SSH 连到这台 Linux。  
项目在 nanocode 里保持选**本地**即可（不选 SSH），因为对 nanocode 来说它和代码在同一台机器上。

nanocode 运行在 Linux 的 **3001** 端口（3000 被其他服务占用）。

- **在 Linux 上**（例如 SSH 进去后 `curl` 或本机浏览器）：<http://localhost:3001>
- **在你 Mac 上打开页面**（二选一）：
  1. **SSH 端口转发**（推荐）：在 Mac 终端执行并保持连接  
     `ssh -L 3001:localhost:3001 你的用户@Linux的IP或主机名`  
     然后在 Mac 浏览器打开：<http://localhost:3001>
  2. **同一内网 / 飞连**：Mac 和 Linux 互通时，浏览器直接打开：<http://10.18.8.55:3001>（若 Linux IP 不同，用 `hostname -I` 在 Linux 上查）

手机连飞连后同样打开：<http://10.18.8.55:3001>（或当前 Linux 的实际 IP）。

以后在 Linux 上启动：`cd nanocode && PORT=3001 npm run dev` 或 `PORT=3001 npm start`。

## 一、基本配置

### 1. 安装与启动

在你已经 SSH 上去的那台机器上（或本机）执行：

```bash
cd /path/to/nanocode   # 例如 /storage/home/zhiningjiao/code/nanocode
./install.sh           # 首次安装：装 Node、依赖等
# 若已装过，直接：
npm run dev            # 开发模式（改代码会重启）
# 或
npm start              # 前台运行
```

服务默认监听 **0.0.0.0:3000**，本机和同网设备都能访问。

### 2. 数据与配置存放位置

- **数据文件**：`nanocode/data/nanocode.json`
  - 项目列表（本地路径、SSH 远程配置）
  - 设置（如 CLI Provider）
  - Session 元数据（归档、管理的会话）

无需单独“配置文件”，在网页里操作即可；Settings 会通过 API 写入上述 JSON。

### 3. 网页内配置项

- **Settings（设置）**
  - **CLI Provider**：选右侧 AI 终端用哪个命令行工具
    - **Claude Code**：`claude`（需已安装 [Claude Code](https://claude.com/code) CLI）
    - **Cursor Agent**：`agent`（需已安装 Cursor 的 agent CLI）
    - **OpenCode**：`opencode`
  - 选好后点 **Save**。  
  **注意**：你当前用的那台机器（或 SSH 目标机）上必须已安装对应命令（`claude` / `agent` / `opencode`），否则右侧 AI 终端会起不来。

- **项目（侧边栏 + 号）**
  - **本地项目**：选一个目录作为工作目录（bash 和 AI 都在这个目录下执行）。
  - **远程（SSH）**：勾选 “Remote (SSH)” 后填：
    - Host、User、Port（默认 22）、工作目录
    - 可选：SSH 私钥路径（如 `~/.ssh/id_ed25519`）  
  这样 nanocode 会通过 SSH 在那台远程机上开 bash 和 AI 会话；远程机上也要装好对应的 CLI。

## 二、在手机上使用（你已能 SSH 到机器）

思路：**在 SSH 那台机器上跑 nanocode**，然后让手机浏览器能访问到 3000 端口。

### 方式 A：手机和运行 nanocode 的机器在同一局域网

例如 nanocode 跑在你实验室/家里的服务器上，手机连的是同一 WiFi：

1. 在服务器上启动 nanocode：`npm run dev` 或 `npm start`。
2. 在服务器上查本机 IP（例如）：
   ```bash
   hostname -I | awk '{print $1}'
   # 或
   ip addr
   ```
3. 手机浏览器打开：`http://<服务器IP>:3000`  
   例如：`http://192.168.1.100:3000`。

若打不开，多半是服务器防火墙拦了 3000 端口，需要在服务器上放行，例如：

```bash
# 若用 ufw（Ubuntu 常见）
sudo ufw allow 3000/tcp
sudo ufw reload
```

### 方式 B：手机和服务器不在同一网（例如服务器在内网/机房）

这时手机无法直接访问服务器 IP，需要用你**能 SSH 到服务器的那台电脑**做“桥梁”：

1. 在**服务器上**先启动 nanocode（SSH 上去执行 `npm run dev` 或 `npm start`）。
2. 在你**自己的电脑**上做 SSH 本地端口转发，把本机的 3000 转发到服务器的 3000：
   ```bash
   ssh -L 3000:localhost:3000 你的用户@服务器地址
   ```
   保持这个 SSH 连接不要关。
3. 手机和电脑在同一 WiFi 下时，手机浏览器访问：**你电脑的 IP:3000**  
   例如：`http://192.168.1.50:3000`（电脑 IP 可在电脑上查：`hostname -I` 或 `ipconfig`）。

这样流量路径是：手机 → 电脑:3000 → SSH 隧道 → 服务器:3000 → nanocode。

### 方式 C：服务器有公网 IP 或做了端口映射

若 3000 端口已在路由器/防火墙上映射到服务器，或服务器本身有公网 IP 且防火墙允许 3000：

- 手机浏览器直接访问：`http://<公网IP或域名>:3000`。  
建议只在可信网络下用，或配合 VPN/反向代理 + HTTPS 使用。

## 三、小结

| 场景 | 操作 |
|------|------|
| 配置 | 网页 Settings 选 CLI Provider；侧边栏添加本地/SSH 项目 |
| 数据 | 自动存在 `data/nanocode.json`，无需手写配置文件 |
| 手机同 WiFi 访问服务器 | 服务器上 `npm run dev`，手机打开 `http://<服务器IP>:3000` |
| 手机不能直连服务器 | 电脑上 `ssh -L 3000:localhost:3000 user@server`，手机访问 `http://<电脑IP>:3000` |

你在手机上用浏览器打开上述地址后，使用方式和电脑一致：左侧 bash、右侧 AI、侧边栏切换项目、Settings 里改 CLI。
